AWS Lambda with CLI

Create and test lambda function

Creating the deployment package on a Mac machine is not likely to work because the execution environment is too different. Amazon provides a machine image for this purpose.  Packages have to be installed in the top directory (with -t option on pip install) and the entire directory subtree has to be in the zip file (with -r option to zip) 

One IAM role can be reused for any simple lambda function. This one was made by doing the preliminary AWS Lambda tutorial at using the console and accepting all defaults.   
# On Amazon image AMI amzn-ami-hvm-2017.03.1.20170812-x86_64-gp2 (ami-4fffc834)
sudo yum update
pip install h5pyd -t ./lambda-six
cd lambda-six
vim lambda-six.py # see file below
zip -r ../lambda-six.zip *
aws configure # enter keys at prompt
aws lambda create-function --function-name lambda-six --region us-east-1 --role arn:aws:iam::111058800152:role/service-role/lambdaExecRole --zip-file fileb:///home/ec2-user/lambda-six.zip --handler lambda-six.my_handler --runtime python2.7 --timeout 15 --memory-size 512 # Note: python2.7
# On iMac execute:
aws lambda invoke --region us-east-1 --payload '{ "domain" : "testfile.testdir.testh5serv.org", "endpoint" : "http://34.230.58.125:5000", "path" : "grp2/grp2b/data3" }' --function-name lambda-six lambda-six.out

Python Handler

Lambda requires a python handler with parameters (event, context). This is a tiny experimental function that computes a sum over a three-dimensional subarray. 
import h5pyd as h5py # imports numpy as dependency
def my_handler(event, context):
domain = event['domain']
endpoint = event['endpoint']
path = event['path']
response = dict(domain=domain, endpoint=endpoint, path=path)
f = h5py.File(domain, 'r', endpoint) # h5pyd File object
d = f[path] # h5pyd Dataset object
X = event['indices'].split(',')
if len(X) != len(d.shape):
response['error'] = 'wrong dimensions'
return response
Y = tuple([slicify(x) for x in X]) # parse indices into slices
if not Y:
response['error'] = 'slice error'
return response
# compute sum of elements in slices
subset = d[Y]
n = reduce(lambda x,y: x*y, subset.shape)
bigsum = sum(subset.reshape((n)))
response['result'] = bigsum
return response
# h5pyd requires subscripts of type numpy.slice or convertible to
# a numpy.slice. This routine parses a string to a numpy.slice.
# Minimal error checking. Real error checking done in calling R package

def isidx(s):

ss = int(s)

E = expr.split(':')
if len(E) != 3:
return None
F = [isidx(e) for e in E]
if not all(F):
return None
return slice(int(E[0]), int(E[1]), int(E[2]))
# Lambda does not use this, but it's useful for testing the handler locally.
# Endpoint is Public IP address, which changes every time the server is restarted.
if __name__ == "__main__":

Input File:

The endpoint is the Public IP of the server instance. The indices need  to be comma-separated slice descriptors. Each slice descriptor has to be three colon-separated integers in the format start:stop:step in python convention, i.e., first array index is 0 and if (start-stop) is a multiple of step, stop is not in the index set. 
{
"domain" : "testfile.testdir.testh5serv.org",
"endpoint" : "http://XXX.XXX.XXX.XXX:5000",
"path" : "grp2/grp2b/data3",
"indices" : "0:10:2,0:4:2,0:4:2"
}
